## 1. HTTPThread — dedicated-thread MiniEventLoop

### Thread + loop creation
- Struct: `HttpThread` — `HTTPThread.rs:86-148`. Fields `loop_: *const MiniEventLoop<'static>` and `uws_loop: *mut uws::Loop` (split out "so HTTPContext can `SocketGroup::init` without naming MiniEventLoop" — `HTTPThread.rs:90-92`).
- Init: `init()` → `_event_loop_draft::init_once` (`HTTPThread.rs:1233-1256`): writes the global `HTTP_THREAD` static *before* spawning (so `on_start` can fill it), sets `HTTP_THREAD_INIT` (Release), spawns a detached named thread ("HTTP Client") whose `JoinHandle` is parked in a static purely for LSAN reachability (`HTTPThread.rs:1218-1227`).
- `on_start` (`HTTPThread.rs:1258-1336`): calls `mini_event_loop::init_global(None, None)` which — critically — does `internal_loop_data.set_parent_raw(2 /* mini */, mini_ptr)` on the uSockets loop; without it the macOS DNS cache-miss path (`dns::getaddrinfo` → `(*loop).internal_loop_data.get_parent()`) panics (`HTTPThread.rs:1274-1279`). Then `thread.http_context.init()`, HTTPS context init eager (if `--ca/--cafile` given, so a bad CA crashes at thread start) or **lazily deferred** to first SSL connect (`lazy_https_init`, `HTTPThread.rs:95-107, 1309-1331`) — saves ~0.7 ms/~400 KB root-CA parse for warm `bun install`.
- Publication: `thread.has_awoken.store(true, Release)` publishes `uws_loop` to cross-thread `wakeup()` readers (`HTTPThread.rs:1332-1334`).
- Loop body: `process_events` (`HTTPThread.rs:1339-1380`) — `num_polls = max(2)` on unix / `inc()` on windows to keep the loop alive with zero sockets, then forever: check `SHUTDOWN_REQUESTED`; `drain_events()`; `uws_loop.inc(); uws_loop.tick(); uws_loop.dec()`. Debug invariant `assert_abort_tracker_sockets_alive()` runs **before and after each tick** (ASAN-poison check on every tracked socket, `HTTPThread.rs:1192-1201`).

### Wakeup / defer
- `wakeup()` (`HTTPThread.rs:1071-1083`): Acquire-load of `has_awoken`, then **raw FFI `us_wakeup_loop(self.uws_loop)`** — deliberately not `Loop::wakeup(&mut self)` because "this runs cross-thread while the HTTP thread owns the loop, so forming `&mut Loop` here would alias". This is the single cross-thread loop primitive the whole design depends on.
- Deferred derefs on-thread: `queued_threadlocal_proxy_derefs: Vec<*mut ProxyTunnel>` (`HTTPThread.rs:133`), pushed by `schedule_proxy_deref` (HTTP-thread-only, `HTTPThread.rs:1007-1011`), drained at the top of `drain_events` (`HTTPThread.rs:867-870`) — the "don't free inside the callback" defer idiom.

### Cross-thread queues — what's queued and how sockets are referenced
Sockets are **never referenced cross-thread**. Every JS→HTTP message carries only a `u32 async_http_id`; the HTTP thread maps id → socket via the thread-local abort tracker (`ArrayHashMap<u32, uws::AnySocket>`, `lib.rs:936-938, 1164-1168`).

| Queue | Element | Lock | Producer API | Drain |
|---|---|---|---|---|
| `queued_tasks` | `NonNull<AsyncHttp<'static>>` (intrusive MPSC node) | lock-free `UnboundedQueue` | `HttpThread::schedule(Batch)` `HTTPThread.rs:1090-1129` — **the only cross-thread entry point** besides wakeup | `drain_events` `HTTPThread.rs:930-953` |
| `deferred_tasks` | `NonNull<AsyncHttp>` parked at capacity | none (HTTP-thread-only) | — | `HTTPThread.rs:908-928` |
| `queued_shutdowns` | `ShutdownMessage { async_http_id }` | `Mutex` | `schedule_shutdown[_by_id]` `HTTPThread.rs:971-983` | `drain_queued_shutdowns` `HTTPThread.rs:634-705` |
| `queued_writes` | `WriteMessage { async_http_id, kind: Data\|End }` | `Mutex` | `schedule_request_write` `HTTPThread.rs:996-1005` | `drain_queued_writes` `HTTPThread.rs:707-765` |
| `queued_receive_resumes` | `u32` (dedup: skips if same as last) | `Mutex` | `schedule_receive_resume` `HTTPThread.rs:960-969` | `HTTPThread.rs:812-854` |
| `queued_cert_check_resumes` | `CertCheckResumeMessage { async_http_id }` (JS `checkServerIdentity` approved) | `Mutex` | `schedule_cert_check_resume` `HTTPThread.rs:985-994` | `HTTPThread.rs:767-810` |

Drain ordering is load-bearing (`HTTPThread.rs:856-865`): resumes → writes → shutdowns → cert-check resumes, so "an abort or cert-rejection scheduled in the same JS turn removes the abort-tracker entry first, so the resume becomes a no-op and the request is never transmitted after a same-tick abort".

Each drain looks up the id in the abort tracker, matches on `uws::AnySocket::{SocketTls,SocketTcp}`, reads the socket's ext tagged pointer (`get_tagged_from_socket`), and dispatches to `HTTPClient` / `h2::ClientSession` / falls through to h3 (`h3::ClientContext::*_by_http_id`). Shutdowns for ids with no socket also scan `pending_h2_connects` waiters (`HTTPContext.rs:416-439`) and h3, else set `has_pending_queued_abort` so the at-capacity fast path still fail-fasts aborted queued tasks (`HTTPThread.rs:115-121, 877-907`).

### Task hand-off (HTTP-thread working copy)
`start_queued_task` (`HTTPThread.rs:1150-1177`): the JS-owned `AsyncHttp` is **bitwise `ptr::read`-copied** into a heap `ThreadlocalAsyncHttp`, tracked in `thread.in_flight`; `cloned.async_http.real = NonNull::new(http)` back-links the original. UAF comment quoted verbatim:

> "Clear stale queue pointers - the clone inherited http.next and http.task.node.next which may point to other AsyncHTTP structs that could be freed before the callback copies data back to the original. If not cleared, retrying a failed request would re-queue with stale pointers causing use-after-free." (`HTTPThread.rs:1170-1173`)

Result delivery back to JS: `AsyncHTTP::on_async_http_callback_raw` (`AsyncHTTP.rs:714-872`) runs **on the HTTP thread**, copies results into the original, and on `!has_more` tears down only clone-owned fields, then `std::alloc::dealloc`s the clone storage — explicitly *not* `Box::drop`, because owned fields (`request_headers`, `header_entries`, `tls_props`, …) are bitwise-shared with the JS-thread original and dropping them "would drop the bitwise-shared fields … and double-free with the JS-thread original" (`AsyncHTTP.rs:770-779, 851-858`). The `callback.run(...)` enqueues onto the JS event loop (result callback owned by the JS side).

### Shutdown
- `shutdown_for_exit()` (`HTTPThread.rs:1415-1468`), called from the JS thread in `global_exit()`: sets `SHUTDOWN_REQUESTED`, `wakeup()`, waits on condvar ≤1 s. HTTP thread's loop sees the flag, runs `dealloc_in_flight_for_exit` (`HTTPThread.rs:1025-1069` — reclaims every in-flight clone box, running only `release_at_shutdown`, not the full result callback), acks, then **parks forever** ("so the loop never ticks the (now partially-freed) sockets again", `HTTPThread.rs:1358-1363`).
- On timeout: "the HTTP thread may still be inside `tick()` and could touch parked allocations. Leak them — the process is exiting and a leak beats a use-after-free." (`HTTPThread.rs:1453-1457`)
- `defer_shutdown_reclaim(ctx, drop_fn)` (`HTTPThread.rs:1401-1409`): parks allocations "whose teardown is not safe while a `tick()` is still on the HTTP-thread stack"; drained on the JS thread after the ack.

## 2. HTTPContext — SocketGroup + keepalive pool

### SocketGroup creation
- `HTTPContext<const SSL: bool>` embeds `group: uws::SocketGroup` — "Embedded sweep/iteration list-head for every socket this context owns (active clients + pooled keepalive). Address-stable" (`HTTPContext.rs:36-38`).
- Plain-TCP + default-TLS: `init()` (`HTTPContext.rs:547-567`) — `group.init(http_thread().uws_loop(), None, owner_ptr)` with `owner_ptr = self as *mut c_void` (group ext = context backpointer); for SSL builds an `SSL_CTX` via `BunSocketContextOptions { request_cert: 1, reject_unauthorized: 0 }.create_ssl_context(&mut err)` then `ssl_ctx_setup` — verification is *always manual* in the handshake callback.
- Default HTTPS with user CA: `init_with_thread_opts` (`HTTPContext.rs:524-545`) maps `create_bun_socket_error_t::{load_ca_file, invalid_ca_file, invalid_ca}` → `InitError` (`HTTPContext.rs:499-522`).
- **Custom-SSL contexts per config**: yes. `HttpThread::connect::<true>` (`HTTPThread.rs:493-587`) keeps a per-thread cache `CUSTOM_SSL_CONTEXT_MAP: ArrayHashMap<*const SSLConfig, SslContextCacheEntry>` keyed by **interned SSLConfig pointer** ("pointer equality is sufficient for lookup", `HTTPThread.rs:23-25`), max 60 entries, 30-min TTL, LRU eviction (`HTTPThread.rs:63-64, 606-620, 1133-1148`). Each entry is a heap `NewHttpContext<true>` (own `SocketGroup` + own `SSL_CTX`) built by `init_with_client_config` → `SSLConfig::as_usockets_for_client_verification()` (`HTTPContext.rs:483-497`; `ssl_config.rs:224-231` forces `request_cert=1, reject_unauthorized=0`). Refcounting: cache holds 1 intrusive ref; each in-flight client holding `client.custom_ssl_ctx` holds 1, "so deinit() never runs while a request is mid-flight" (`HTTPContext.rs:26-31`).
- Context teardown (`impl Drop`, `HTTPContext.rs:1066-1117`): drain pooled sockets (force-close, see §8), then `group.close_all()` and `SocketGroup::destroy(&raw mut self.group)` — "Force-close any remaining sockets before unlinking the group so the loop never dereferences a freed `*Context` via `group->ext`" (`HTTPContext.rs:1100-1107`) — then `SSL_CTX_free`. Tolerates a never-init'd group (`group.loop_.is_null()` check, `HTTPContext.rs:1096-1100`).

### Keepalive pool (HiveArray)
- `pending_sockets: HiveArray<PooledSocket<SSL>, POOL_SIZE=64>` (`HTTPContext.rs:19, 33, 66-67`), `MAX_KEEPALIVE_HOSTNAME = 128`.
- `PooledSocket` (`HTTPContext.rs:162-197`) stores the socket **by value as `HTTPSocket<SSL>` (= `uws::SocketHandler<SSL>`, a wrapper over the raw `us_socket_t*`)** plus: inline `hostname_buf[128]`/`hostname_len`, `port`, TLS-strictness flags (`did_have_handshaking_error_while_reject_unauthorized_is_false`, `established_with_reject_unauthorized`), `ssl_config: Option<ssl_config::SharedPtr>` (strong Arc ref while parked), `owner: *mut HTTPContext<SSL>` (return-to-correct-pool), `proxy_tunnel: Option<RefPtr>` (pool owns one strong ref), `target_hostname`/`target_port` (CONNECT upstream), `proxy_auth_hash: u64`, `h2_session: Option<NonNull<h2::ClientSession>>` (pool owns one strong ref).
- **Park** — `release_socket` (`HTTPContext.rs:581-686`): preconditions `!is_closed && !is_shutdown && get_error()==0 && is_established` and hostname ≤128; claims a hive slot, writes the **slot address into the socket's ext** *before* writing the struct (`HTTPContext.rs:614-625`), then `socket.flush(); socket.timeout(0); socket.set_timeout_minutes(5)` — pooled idle timeout is 5 minutes on the long (minute-granularity) wheel. If pooling fails: tunnel `shutdown()+detach_and_deref()`, h2 session deref, `close_socket` (graceful).
- **Key / validation on reuse** — `existing_socket` (`HTTPContext.rs:689-832`) linearly scans the hive matching: port; `ssl_config` by interned-pointer equality; strictness (skip if socket had handshake error under RU=false and caller is strict; skip if caller is strict but socket `!established_with_reject_unauthorized` — for direct TLS *and* tunnel inner-TLS, `HTTPContext.rs:721-781`); ALPN compatibility (`h2_session` presence vs caller's `AlpnOffer`, `HTTPContext.rs:729-736`); tunnel presence must match; `proxy_auth_hash` (covers proxy credentials **and** the Host-header SNI override for direct TLS, `HTTPContext.rs:743-747`, `HTTPContext.rs:942-952`); tunnel target host:port; case-insensitive hostname. Liveness re-check at match time: `is_closed()` → mark dead + skip; `is_shutdown() || get_error()!=0` → `terminate_socket` + skip (`HTTPContext.rs:790-797`).
- On hit: strong refs (ssl_config / tunnel / h2_session) are moved out of the slot, slot returned via `pending_sockets.put`, socket's ext retagged to the new `HTTPClient`, `client.allow_retry = true`; then either h2 re-registration+adopt, tunnel `adopt` + `on_open`/`on_writable`, or plain `on_open` + `first_call` (`HTTPContext.rs:954-1022`).
- **Events on pooled sockets** (ext tag = `PooledSocket`): `on_data` — data on an idle tunneled socket = evict ("inner-TLS traffic … we can't process without the SSLWrapper. We'd hand back a tunnel whose inner state diverged from ours", `HTTPContext.rs:1260-1269`); idle h2 data forwarded to `session.on_idle_data` and evicted if `!can_pool()`; trailing `0\r\n\r\n` of a chunked response ignored; anything else → `terminate_socket` (`HTTPContext.rs:1254-1292`). `on_writable` — no-op (`HTTPContext.rs:1300-1301`). `on_handshake` on a pooled socket (TLS renegotiation-ish): success → re-arm 5-min timer, else terminate (`HTTPContext.rs:1203-1218`). Timeout → falls to `terminate_socket` (client_mut/session_mut both None, `HTTPContext.rs:1309-1320`).
- **Close of a pooled socket**: every dead-marking path funnels through `mark_tagged_socket_as_dead` (`HTTPContext.rs:303-312`) which, if the tag is `PooledSocket`, calls `add_memory_back_to_pool` (`HTTPContext.rs:1236-1252`): `release_parked_refs()` (drop ssl_config Arc, deref tunnel + h2 refs, `HTTPContext.rs:250-264`), then `(*owner).pending_sockets.put(slot)` — then the ext is rewritten to the `DeadSocket` sentinel (a static, `HTTPContext.rs:1390-1402`). `on_close`/`on_connect_error` on a non-client socket also **sweep the abort tracker by socket pointer** as a backstop (`HTTPContext.rs:1229-1234, 1337-1342`; rationale at `lib.rs:1170-1191`: a stale entry "would survive into `us_internal_free_closed_sockets`, leaving `drain_queued_shutdowns` to dereference freed memory on a later abort").

## 3. Vtable slots implemented and every socket-level API used

### Handler vtable (`Handler<SSL>`, `HTTPContext.rs:1121-1386`) — dispatch key is the ext tagged pointer, passed pre-dereferenced (`ptr` = the ext *value*, `HTTPContext.rs:329-333`)
- `on_open` (1124) — client `on_open` or terminate.
- `on_handshake(ptr, socket, success: i32, ssl_error: us_bun_verify_error_t)` (1141) — TLS verify outcome + manual reject logic (see §6).
- `on_close(ptr, socket, code, reason)` (1220).
- `on_data(ptr, socket, &[u8])` (1254).
- `on_writable` (1294).
- `on_timeout` (short tick) and `on_long_timeout` (minute wheel) — both implemented, identical handling, because `HTTPClient::set_timeout` routes to either wheel by duration (1309-1327).
- `on_connect_error(ptr, socket, code)` (1329) — reads `socket.dns_error()` *before* marking dead ("uSockets keeps the connecting socket alive for the whole dispatch"); notes "us_connecting_socket_close is always called internally by uSockets" (1343).
- `on_end` (FIN) (1346-1385) — four-state comment; graceful vs reset close decision quoted in §8.

### `us_socket_*` / handler methods consumed (across HTTPContext.rs, HTTPThread.rs, lib.rs, ProxyTunnel.rs, h2)
- Connect: `HTTPSocket::<SSL>::connect_group(&mut group, SocketKind, Option<*mut SSL_CTX>, hostname, port, ext_value, allow_half_open=false)` (`HTTPContext.rs:1025-1039`) and `connect_unix_group(...)` (`HTTPContext.rs:843-856`). Note the per-call `Option<SSL_CTX>` — the group is shared but the SSL_CTX is supplied per connect.
- Write: `socket.write(&[u8]) -> isize` (`lib.rs:1420`, `ProxyTunnel.rs:497-498, 726`, `h2 ClientSession.rs:671`).
- Flush: `socket.flush()` before parking (`HTTPContext.rs:626`).
- Timeouts: `socket.timeout(secs)`, `socket.set_timeout_minutes(min)`, `socket.set_timeout(secs)` (routes ≤240 s to short-tick timer, >240 s to minute wheel — `lib.rs:575-582, 1145-1155, 4108-4114`), `set_timeout(0)`/`timeout(0)` to disarm.
- Shutdown (half-close after request write when no keepalive): `socket.shutdown()` (`lib.rs:3214, 3246`).
- Close: `socket.close(uws::CloseKind::{Normal,Failure})` everywhere (Normal = SSL clean shutdown; Failure = force/RST — see quotes in §8).
- State queries: `is_closed()`, `is_shutdown()`, `is_established()`, `get_error()`, `is_closed_or_has_error()`.
- Flow control: `socket.pause_stream()` / `socket.resume_stream()` (`lib.rs:4116-4147, 4286-4292`) — driven by JS-side backpressure via `schedule_receive_resume`; **"The uSockets paused bit survives `state.reset()`; never hand a paused socket back to the pool"** (`lib.rs:4286-4287`).
- TLS handle: `socket.get_native_handle() -> *mut SSL` (`HTTPContext.rs:1169-1174`; `lib.rs:1828-1834, 1971-1983` for `SSL_get0_alpn_selected`, SNI/ALPN config, `SSL_is_init_finished`).
- Verify error: delivered as the `us_bun_verify_error_t` argument of `on_handshake` (outer TLS: `HTTPContext.rs:1141-1149`; inner TLS: `ProxyTunnel.rs:364-380`), decoded by `HTTPCertError::from_verify_error` — no separate `us_socket_verify_error` call in this crate.
- DNS: `socket.dns_error()` (`HTTPContext.rs:1332`), raw getaddrinfo code plumbed into results (`lib.rs:2144-2154`).
- Misc: `socket.set_keep_alive(true, 60)` (TCP keepalive, Node/undici parity, `lib.rs:1796-1816`), `socket.fd()` for sendfile (`lib.rs:3405-3406`), `socket.socket` (raw `InternalSocket`) for `AnySocket` construction and pointer-compare sweeps, `SocketTLS::from_any` / `SocketTCP::from_any` / `assume_ssl()` / `cast_ssl::<IS_SSL>()` const-generic bridging.
- Loop: `us_wakeup_loop` raw FFI (`HTTPThread.rs:1081`), `Loop::{tick, inc, dec, num_polls, wakeup}` (`HTTPThread.rs:1339-1372`, `h3 PendingConnect.rs:128`), `internal_loop_data.set_parent_raw` (via MiniEventLoop init).
- SocketGroup: `group.init(loop, None, owner_ptr)`, `group.close_all()`, `SocketGroup::destroy` (`HTTPContext.rs:518-521, 1100-1107`).
- SSL_CTX creation: `BunSocketContextOptions::create_ssl_context(&mut create_bun_socket_error_t)` + `bun_boringssl::ssl_ctx_setup` + `SSL_CTX_free`.

## 4. Ext-data types and SocketKind

- `SocketKind`: `HTTPContext::<SSL>::KIND = if SSL { uws::SocketKind::HttpClientTls } else { uws::SocketKind::HttpClient }` (`HTTPContext.rs:297-301`), passed to every `connect_group`/`connect_unix_group`.
- Ext is exactly one pointer word: a `TaggedPtrUnion` (`ActiveSocket<SSL>`) with 4 members, tags descending from 1024 (`HTTPContext.rs:71-113`):
  - `1024 DeadSocket` (static sentinel), `1023 HTTPClient<'static>`, `1022 PooledSocket<SSL>` (hive slot address), `1021 h2::ClientSession`.
- Write invariant: "the ext slot of every HTTP-thread socket holds exactly the `ActiveSocket` tagged-pointer word; uSockets allocates it as `size_of::<*mut c_void>()` and never reads/writes it itself … `ext()` returns `None` only for closed sockets, in which case the write is a no-op" (`set_socket_ext`, `HTTPContext.rs:343-356`).
- Attach: ext set to the `HTTPClient` pointer at connect time (`HTTPContext.rs:1030-1037, 848-854`) or pool-reuse time (`HTTPContext.rs:970-978`). Detach: retag to hive-slot address on park (`HTTPContext.rs:622-625`), to session pointer on ALPN=h2 (`tag_as_h2`, `HTTPContext.rs:472-474`), to `DeadSocket` on any dead-marking. The `HTTPClient` itself lives inside the HTTP-thread `ThreadlocalAsyncHTTP` clone — no JS object ever aliases the socket.
- Lifetime contract (quoted): "a tagged pointer stored in a live socket's ext slot identifies an object that is alive for the duration of the dispatched callback — `HTTPClient` until its terminal result callback, `h2::ClientSession` while it holds a registry strong ref, `PooledSocket` while its HiveArray bit is set. … Callers … must not retain the returned reference past the callback." (`HTTPContext.rs:120-127`)

## 5. ProxyTunnel — CONNECT + TLS-in-TLS via SSLWrapper

- Type: `ProxyTunnel` (`ProxyTunnel.rs:45-66`), intrusive single-thread refcounted; `wrapper: Option<SSLWrapper<*mut HTTPClient>>` where `SSLWrapper` is `bun_uws::ssl_wrapper::SSLWrapper` (re-exported `lib.rs:70-71`) — a memory-BIO TLS engine, **not** a second uSockets socket. `socket: bun_uws::MaybeAnySocket` (Ssl/Tcp/None) is the *outer* transport (which may itself be TLS → TLS-in-TLS).
- Setup: `ProxyTunnel::start` (`ProxyTunnel.rs:606-668`) — `SSLWrapper::init_from_options(&ssl_config.as_usockets_for_client_verification(), is_client=true, Handlers { on_open, on_data, on_handshake, on_close, write: write_encrypted, on_session: None, on_keylog: None, ctx: *mut HTTPClient })`, then `wrapper.start()` / `start_with_payload(start_payload)`.
- Direct `us_*`/uws usage inside the tunnel:
  - Outer writes: `socket.write(encoded_data)` in `write_encrypted` (`ProxyTunnel.rs:471-508`) with FIFO `write_buffer` ordering preservation, and in `on_writable` (`ProxyTunnel.rs:707-743`) followed by `wrapper.flush()`.
  - Inbound: `HTTPClient::on_data` routes raw bytes into `tunnel.receive(buf)` → `wrapper.receive_data` (`ProxyTunnel.rs:745-759`), which synchronously fires decoded `on_data`/`on_handshake`/`on_close`.
  - `us_bun_verify_error_t` arrives via the wrapper's `on_handshake` (`ProxyTunnel.rs:364-368`) → `HTTPCertError::from_verify_error` → same manual reject / `check_server_identity` flow as the outer socket (`ProxyTunnel.rs:381-468`), including SNI+ALPN(H1-only) config on the inner `SSL*` in `on_open` (`ProxyTunnel.rs:239-266`).
  - Data-stage dispatch reaches back into `http_thread().{https,http}_context` raw pointers (`ProxyTunnel.rs:343-354, 589-601`).
- Deferred deref: `on_close` bumps the refcount via raw `Cell` projection and releases it via `http_thread().schedule_proxy_deref(ptr)` — "Deref after returning to the event loop to avoid lifetime hazards" (`ProxyTunnel.rs:528-536, 552-565, 582-583`).
- Pooling of tunnels: `detach_owner` (`ProxyTunnel.rs:791-806`, "We intentionally leave wrapper.handlers.ctx stale here. The tunnel is idle in the pool and no callbacks will fire until adopt() reattaches a new owner"), `adopt` (`ProxyTunnel.rs:812-843` — resets write_buffer, rebinds `handlers.ctx`, no new CONNECT/handshake, stages set to ProxyHeaders). Poolability gate in `lib.rs:4233-4258` (request side drained, wrapper `!is_shutdown && !fatal_error && !has_pending_data`).
- The whole callback module is governed by the ALIASING NOTE (`ProxyTunnel.rs:215-223`): callbacks are invoked synchronously from inside an SSLWrapper method whose `&mut self` **is** `(*tunnel).wrapper`, so callbacks never materialise `&mut ProxyTunnel` — all field access via `addr_of!` disjoint projections.

## 6. TLS specifics

- **Config model** (`ssl_config.rs`): `SSLConfig` = owned NUL-terminated C-string fields + numeric options, converted 1:1 into `uws::socket_context::BunSocketContextOptions` by `as_usockets()` (`ssl_config.rs:174-219`). Client paths always use `as_usockets_for_client_verification()` → `request_cert=1, reject_unauthorized=0` (`ssl_config.rs:224-231`) — **uSockets never rejects; Bun decides in `on_handshake`**.
- **Interning**: `global_registry::intern` (`ssl_config.rs:535-654`) — content-hash (wyhash, memoized `AtomicU64`) + `Weak` dedup under a mutex; makes *pointer equality* the SSL_CTX cache key and pool key. (Note the registry's own aliasing discipline: probe via `Weak::upgrade`, never a raw-key content compare, `ssl_config.rs:538-553`.)
- **SSL_CTX cache**: §2 (per-interned-config `NewHttpContext<true>`, 60 entries / 30 min TTL / LRU). Default HTTPS context is lazily built (§1).
- **SNI**: hostname priority `tls_props.server_name > client.hostname (Host header, port-stripped) > url.hostname` (`get_tls_hostname`, `lib.rs:1193-1199`); set via `SSL_set_tlsext_host_name` in `configure_http_client_with_alpn` (`lib.rs:1111-1140`), skipped for IP literals (RFC 6066), using a thread-local 8 KB `TEMP_HOSTNAME` scratch. Called from `on_open` on the outer socket (`lib.rs:1827-1866`) and from the tunnel's inner `on_open` (`ProxyTunnel.rs:239-266`). Also sets `SSL_OP_LEGACY_SERVER_CONNECT`, ALPN list (h1 / h2 / h2+h1 per `alpn_offer()`), SCT + OCSP stapling.
- **Verify-error mapping**: `on_handshake` receives `us_bun_verify_error_t {error_no, code, reason}`; `HTTPCertError::from_verify_error` (`HTTPCertError.rs:32-60`) widens `code`/`reason` to `&'static ZStr` (uSockets' `us_ssl_socket_verify_error_str` table and BoringSSL's `X509_verify_cert_error_string` literals — genuinely process-lifetime). Errors become `get_cert_error_from_no(error_no)` failures.
- **reject_unauthorized**: if true and `error_no != 0` → `close_and_fail`; if true and chain OK → `check_server_identity` (`lib.rs:1679-1752`): native SAN check (`check_x509_server_identity`) fast path, or — when JS registered `checkServerIdentity` (`signals CertErrors`) — DER-serialize the leaf, park the connection (`is_waiting_for_cert_check`), deliver to JS, resume via `schedule_cert_check_resume` or abort (`lib.rs:1704-1737`). If RU=false, handshake errors are recorded (`did_have_handshaking_error`) but the connection proceeds; the pool's strictness flags (§2) prevent lax-established sockets/tunnels/h2 sessions from being reused by strict callers (`HTTPContext.rs:749-781, 898-918`; `ProxyTunnel.rs:53-64, 796-802`).
- **Session reuse**: no TLS session-ticket cache — the fetch tunnel explicitly opts out: "fetch's proxy tunnel surfaces no 'session'/'keylog' events; opting out keeps its SSL off the parked queues entirely" (`ProxyTunnel.rs:628-630`). Reuse is connection-level (keepalive pool, h2 multiplexing) only.

## 7. h2_client / h3_client boundaries

### h2 (`h2_client/ClientSession.rs`)
No new `us_*` surface: rides the same `HTTPSocket<true>` created by `HTTPContext<true>` after ALPN selects "h2" (`lib.rs:1971-1998` — `SSL_get0_alpn_selected` on `get_native_handle()`; socket ext retagged via `tag_as_h2`). Socket ops: `socket.write` in `flush()` (loop until short write, `ClientSession.rs:657-687`), `socket.set_timeout(want)` in `rearm_timeout` (max effective idle timeout across streams, global floor when any stream is unbounded, `ClientSession.rs:520-550`), `socket.is_closed_or_has_error()`, `sock.close(CloseKind::Failure)` after best-effort GOAWAY in `fail_all` (`ClientSession.rs:845-860`), and pool interaction through `HTTPContext` (`register_h2`/`unregister_h2_raw`/`maybe_release`, `ClientSession.rs:917-945` — note the re-entrancy contract: `unregister_h2_raw` uses raw place projection because `connect() → adopt()` may still hold `&mut HTTPContext<true>`, `HTTPContext.rs:441-470`). Session is the socket-ext object; per-request routing is `async_http_id` scans (`abort_by_http_id`, `stream_body_by_http_id`, `resume_receive_by_http_id`, `ClientSession.rs:864-899, 552-580`).

### h3 (`h3_client/*`) — the lsquic/uSockets-QUIC boundary
Everything goes through `bun_uws::quic` (the `us_quic_*` C layer over lsquic + a uSockets UDP endpoint). Required APIs:
- `quic::global_init()` once (`ClientContext.rs:40, 77`).
- `quic::Context::create_client(loop_ptr, 0, ext_size_conn = sizeof(*mut ClientSession), ext_size_stream = sizeof(*mut Stream))` → `us_create_quic_socket_context` (`ClientContext.rs:79-87`). Engine is **bound to the loop** ("it lives on `loop->data.quic_head` and is driven by that loop's pre/post hooks" — `ClientContext.rs:28-33`); one engine per loop, process-global only because the HTTP thread is a singleton.
- `callbacks::register(qctx)` installs C callbacks: `on_hsk_done`, `on_goaway`, `on_conn_close`, `on_stream_open`, `on_stream_headers`, `on_stream_data`, `on_stream_writable`, `on_stream_close` (`callbacks.rs:77-297`), all invoked on the HTTP thread from lsquic's process_conns.
- `qctx.connect(host_z, port, sni, reject_unauthorized, ext) -> ConnectResult::{Socket(*mut quic::Socket), Pending(*mut quic::PendingConnect), Err}` (`ClientContext.rs:142-181`); underlying `us_quic_connect_addr` (`h3 ClientSession.rs:32, 74-75`).
- Ext slots on `quic::Socket` (→ `*mut ClientSession`) and `quic::Stream` (→ `*mut Stream`) — same pattern as TCP ext (`ClientContext.rs:152`, `callbacks.rs:47-71`).
- DNS-pending path needs loop plumbing: `qctx.r#loop()`, `bun_dns::internal::register_quic`, and the thread-safe DNS completion pushes onto a list and calls raw `(*loop_ptr).wakeup()` — "us_internal_dns_callback_threadsafe: push onto a mutex-protected list and [wake]" (`PendingConnect.rs:55-128`); resolved entries drained by `h3::PendingConnect::drain_resolved()` from `HTTPThread::drain_events` (`HTTPThread.rs:865`).
- Cross-thread JS→h3 routing again by `async_http_id`: `ClientContext::{abort,stream_body,resume_receive}_by_http_id` (`ClientContext.rs:198-238`), reached as the abort-tracker-miss fallback in the HTTPThread drains (`HTTPThread.rs:687, 755, 850`). h3 sessions **never appear in the abort tracker** ("an HTTP/3 session … has no TCP socket to register in the tracker", `HTTPThread.rs:685-687`).
- Eligibility gate `can_try_h3_alt_svc` (`lib.rs:1921-1953`): h3 bypasses `check_server_identity`, custom request ctx, proxies, unix sockets, sendfile.

## 8. Unusual lifecycle — verbatim UAF-discipline quotes

- **Pool-owned sockets with no JS object**: `PooledSocket` in the hive; the only owner reference is the socket ext word pointing at the hive slot and the slot's `owner: *mut HTTPContext`. Nothing on the JS side knows the socket exists.
- **"Closed but in pool"**: reuse revalidates (`is_closed`/`is_shutdown`/`get_error`, `HTTPContext.rs:790-797`); close of a parked socket recycles the slot via `mark_tagged_socket_as_dead` → `add_memory_back_to_pool` before the tag becomes `DeadSocket`.
- Context Drop, pooled-tunnel eviction (`HTTPContext.rs:1080-1087`):
  > "Do NOT call rp.data.shutdown() here — it drives SSLWrapper.shutdown → triggerCloseCallback → onClose(handlers.ctx), and handlers.ctx is the stale HTTPClient pointer from detachOwner(). That client is freed by now. http_socket.close(.failure) below force-closes the TCP without triggering the callback, same as addMemoryBackToPool()."
- Context Drop, force-close rationale (`HTTPContext.rs:1068-1072`):
  > "Must force-close (code != 0) because SSL clean shutdown (code=0) requires a shutdown handshake with the peer, which won't complete during eviction. Without force-close, the socket stays linked and the context refcount never reaches 0, leaking the SSL_CTX."
- Abort-tracker staleness backstop (`lib.rs:1172-1177`):
  > "when `Handler::on_close` fires on a socket whose ext has already been retagged to `DeadSocket`/`PooledSocket`, the client/session dispatch is skipped and any stale entry would survive into `us_internal_free_closed_sockets`, leaving `drain_queued_shutdowns` to dereference freed memory on a later abort."
- Debug invariant (`HTTPThread.rs:1186-1191`):
  > "every socket pointer in the abort tracker must point at live (unfreed) memory. A stale entry here means some socket-close path forgot `unregister_abort_tracker`, which later manifests as a use-after-free in `drain_queued_shutdowns`/`drain_queued_writes` when the JS thread aborts that request id."
- Callback-may-free-client discipline (`HTTPContext.rs:1176-1183`):
  > "checkServerIdentity already called closeAndFail() → fail() → result callback, which may have destroyed the AsyncHTTP that embeds `client`. Socket is terminated and the abort tracker is unregistered there, so the only safe action is to return without touching `client` again."
  (same shape in `ProxyTunnel.rs:419-424, 432` and in `drain_queued_cert_check_resumes`, `HTTPThread.rs:784-787`: "May synchronously reach close_and_fail → dispatch_result_and_reset; do not touch `client` after this call.")
- Close-before-open race (`HTTPThread.rs:652-655`):
  > "If we only call socket.close(), then it won't call `onClose` if this happens before `onOpen` is called." (hence `close_and_abort` on connecting sockets in the shutdown drain.)
- FIN handling policy on `on_end` (`HTTPContext.rs:1361-1366`):
  > "A FIN that cuts the request short while its body is still being sent is answered with a reset instead: a graceful close would queue our FIN behind the not-yet-delivered body bytes (a server that rejects an upload early stops reading them), so the peer would never observe the connection closing and it would leak."
- Deferred tunnel deinit inside callbacks: "Drop our temporary ref asynchronously to avoid freeing within callback" (`ProxyTunnel.rs:552-553`); "Deref after returning to the event loop to avoid lifetime hazards" (`ProxyTunnel.rs:582`).
- Cross-thread enqueue of writes/results: all JS→HTTP traffic is id-based messages + `us_wakeup_loop` (§1); HTTP→JS is the result callback on the clone, with the bitwise-shared-fields dealloc discipline quoted in §1/§4 (`AsyncHTTP.rs:770-779, 851-857`).
- Exit teardown: quotes in §1 (`park forever`, `a leak beats a use-after-free`, `defer_shutdown_reclaim`).

## 9. Migration notes — what a Rust rewrite of the uSockets core must provide

1. **Loop-local sockets + one thread-safe wakeup are sufficient — and assumed.** Nothing dereferences a socket cross-thread; the only cross-thread primitives used are `us_wakeup_loop(*mut Loop)` (callable without `&mut Loop` — the aliasing note at `HTTPThread.rs:1078-1081` is explicit) and the MPSC/mutex queues in HTTPThread. The new API must expose a wakeup that is safe while the loop-owning thread is inside `tick()`, without forming a unique reference to the loop.
2. **Stable socket identity across callbacks and while parked.** The keepalive pool stores `HTTPSocket` values (raw socket pointers) for up to 5 minutes and the abort tracker maps `u32 → AnySocket` across ticks. uSockets' current guarantee — the `us_socket_t*` stays valid until after `on_close` dispatch, with `ext()` readable during close and connecting sockets alive through `on_connect_error` (`HTTPContext.rs:1330-1331`) — must be preserved, or replaced with a generation-checked handle. If handles can dangle after close (today they can, hence `assert_abort_tracker_sockets_alive` and the by-pointer tracker sweep), a safe-handle design would delete an entire bug class.
3. **One pointer-word ext per socket, caller-owned**, plus per-group ext (context backpointer) and per-loop parent pointer (`set_parent_raw` used by DNS). The dispatcher must pass the ext *value* to callbacks (`HTTPContext.rs:329-333` relies on this).
4. **SocketGroup semantics**: embedded, address-stable list-head; `init(loop, ssl_ctx?, ext)`, `close_all()`, `destroy()` (unlink before free); *per-connect* SSL_CTX override (`connect_group(..., Option<SSL_CTX>, ...)`) so one group can serve multiple SSL_CTXs is *not* required — but per-context SSL_CTX with cheap group creation is (custom-SSL cache creates up to 60 groups).
5. **Callback set required**: open, data, writable, close(code,reason), end(FIN half-close, distinct from close), connect_error(with dns_error readable), handshake(success, verify_error{error_no, code:*const char, reason:*const char} with static-lifetime strings — `HTTPCertError.rs:38-49` widens them to `'static`), short-tick timeout **and** minute-granularity long timeout (two wheels; `set_timeout` routes at 240 s, `lib.rs:579-581`).
6. **Socket ops required**: write (partial-write count), flush, shutdown (write-side), close(normal|failure — failure must skip the TLS shutdown handshake and must not fire the SSLWrapper-style close path; see §8 quotes), pause_stream/resume_stream (**paused bit must be introspectable or reset on reuse** — `lib.rs:4286-4291`; resume must re-arm delivery of latched FIN/RST/error, `lib.rs:4139-4144`), is_closed/is_shutdown/is_established/get_error, set_keep_alive(idle secs), fd() (sendfile), get_native_handle() → `SSL*` (the client directly calls BoringSSL for SNI/ALPN/OCSP/peer-chain — the core must not hide the handle), unix-socket connect, no-half-open flag on connect.
7. **Manual verification mode**: `request_cert=1, reject_unauthorized=0` context creation with structured CA errors (`create_bun_socket_error_t` triple), handshake callback delivering the verify error rather than tearing the socket down — Bun's whole cert-error/`checkServerIdentity`/lax-reuse machinery depends on the connection surviving a failed verification.
8. **Re-entrancy discipline**: callbacks synchronously re-enter (write from within data, close from within handshake, connect from within close). Handler dispatch must tolerate the ext being retagged mid-callback and must never cache `&mut` state across a user callback. Terminal callbacks may free the ext object (HTTPClient) — the dispatcher must not touch ext after invoking them.
9. **Loop keep-alive controls**: `inc`/`dec`/`num_polls` are used to keep the loop polling with zero sockets (`HTTPThread.rs:1341-1348`); a rewrite needs an equivalent "hold the loop alive" refcount.
10. **QUIC/UDP for h3**: keep the `us_quic` surface: `create_client(loop, ext sizes)`, engine driven by the loop's pre/post hooks, `connect(host, port, sni, reject, ext)` with sync/pending(DNS)/err tri-state, conn+stream ext slots, thread-safe DNS completion + loop wakeup, and the callback set in §7. Everything above the `quic::*` module is lsquic-agnostic already.
11. **Exit story**: today's contract is "wake, drain, ack, park forever; on timeout leak rather than free" (§1/§8). A rewrite could improve this (cancellable tick / closeable loop), but must at minimum allow the ack-and-park pattern without the loop touching sockets again.
12. **Timers on pooled sockets**: parked sockets rely on `timeout(0) + set_timeout_minutes(5)`; the long wheel must survive with no active request object attached.
