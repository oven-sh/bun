## Macro-generated vs source-direct

- **Source-direct unsafe**: ~85 % of B's unsafe surface — `unsafe { ... }` blocks written inline in human-authored code with paired `// SAFETY:`.
- **Macro-generated unsafe**:
  - `#[bun_jsc::JsClass]` / `#[bun_jsc::host_fn]` — emits `extern "C"` shim per JS-exposed method (~250 invocations across section). The macro-emitted body inserts `bun_jsc::to_js_host_call(g, || …)` which discharges exception-scope preconditions.
  - `export_callbacks!` / `export_lazy_prop_callbacks!` (BunObject.rs) — stamps ~65 `#[unsafe(no_mangle)] pub unsafe fn` shims with `jsc_host_abi!` for Windows-x64 SysV.
  - `bun_uws::uws_callback(thunk = "…_c")` — emits the `unsafe extern "C" fn …_c(ctx, …)` thunk paired with a safe wrapper.
  - `bun_event_loop::impl_timer_owner!` — emits the `from_timer_ptr` + intrusive-link glue (2 invocations: CronJob, Subprocess).
  - `bun_spawn::link_impl_ProcessExit!` — emits `on_process_exit` fn-ptr arm (2 invocations: CronRegisterJob + Subprocess).
  - `#[derive(bun_ptr::CellRefCounted)]` / `RefCounted` — emits the `Drop`-equivalent `destroy` glue.
  - `#[derive(bun_jsc::JsClass)]` codegen (separate path) — emits the C-ABI `…__construct`/`…__finalize` shims tied to `cron.classes.ts`/`Glob.classes.ts`/etc.

4. **`js_bundle_completion_task.rs`** — `deinit(this: *mut Self)`, `on_complete_anytask(ctx: *mut Self)`, `unsafe fn link(item: *mut Self)`.
5. **`Archive.rs` AsyncTask** — `schedule(this: *mut Self)`, `promise_value(this: *mut Self)`, `run_callback(work_task: *mut WorkPoolTask)` recovers `*mut Self` via `from_field_ptr!`.
6. **`bun/subprocess/SubprocessPipeReader.rs`** — `start_with_keepalive` derives `*mut Self` via `std::ptr::from_mut(self)` and feeds to `ScopedRef::new`.
7. **`filesystem_router.rs` MatchedRoute** — `deinit(this: *mut MatchedRoute)` and `finalize(self: Box<Self>) { Self::deinit(Box::into_raw(self)) }`.

In every case the body either keeps everything raw, or forms a short-lived `&Self` (shared) reborrow whose lifetime ends before any re-entrant call. **No `&mut self` is ever held across a callback that may free `self`** — discipline matches Section F precisely.

## FFI calling-convention contracts (Glob/Archive/BunObject/JSBundler)

- **BunObject.rs** — `jsc_host_abi!` macro selects `extern "sysv64"` on Windows-x64 (the C++ side is declared `JSC_HOST_CALL_ATTRIBUTES`/`SYSV_ABI`); plain `extern "C"` would be MS-x64 there and put `globalObject` in RCX vs C++'s RDI → garbage deref. The block comment at 268–273 cites the bug-class explicitly. Confirmed across all 32 callback exports.
- **JSBundler.rs** — `JSBundlerPlugin__create`/`tombstone`/`runOnEndCallbacks`/`appendDeferPromise`/`setConfig`/`runSetupFunction`/`loadAndResolvePluginsForServe` all use `safe fn` inside `unsafe extern "C"` (Plugin handle is `opaque_ffi!` ZST; validity proof is in the type). Only `JSBundlerPlugin__create` returns a raw `*mut Plugin`.
- **Glob.classes.ts** — JsClass codegen emits the C ABI; no hand-written FFI in `glob.rs` itself.
- **Archive.rs** — libarchive FFI lives in `src/libarchive_sys` (section T); Archive.rs uses safe Rust wrappers over those.
