# Cookies

Bun provides a built-in API for managing cookies via the `CookieMap` class. This API is accessible through the `cookies` property on the `BunRequest` object. When using the `routes` property in `Bun.serve()`, any changes made to `request.cookies` are automatically synchronized with the outgoing `Response` headers.

## The CookieMap API

The `CookieMap` behaves similarly to a standard JavaScript `Map`, but it is specifically optimized for handling cookie strings and `Set-Cookie` attributes.

### Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `get` | `(name: string) => string \| undefined` | Retrieves the value of a cookie by name. |
| `set` | `(name: string, value: string, options?: CookieOptions) => this` | Adds or updates a cookie. |
| `has` | `(name: string) => boolean` | Checks if a cookie exists in the request. |
| `delete` | `(name: string, options?: CookieOptions) => boolean` | Sets a `Set-Cookie` header with an expired date to remove the cookie from the client. |
| `entries` | `() => IterableIterator<[string, string]>` | Returns an iterator of all cookie name-value pairs. |
| `keys` | `() => IterableIterator<string>` | Returns an iterator of all cookie names. |
| `values` | `() => IterableIterator<string>` | Returns an iterator of all cookie values. |

## Reading cookies

Cookies are automatically parsed from the `Cookie` header of incoming requests.

```ts
Bun.serve({
  routes: {
    "/profile": (req) => {
      const sessionId = req.cookies.get("session_id");

      if (!sessionId) {
        return new Response("Unauthorized", { status: 401 });
      }

      return Response.json({ sessionId });
    },
  },
});
```

## Setting cookies

Use `req.cookies.set()` to define cookies. Bun automatically generates the appropriate `Set-Cookie` header.

### Cookie Options

The `set` and `delete` methods accept an optional configuration object:

| Option | Type | Description |
|--------|------|-------------|
| `domain` | `string` | The domain for which the cookie is valid. |
| `expires` | `Date` | The absolute expiration date of the cookie. |
| `httpOnly` | `boolean` | If `true`, the cookie is inaccessible to client-side scripts. |
| `maxAge` | `number` | Number of seconds until the cookie expires. |
| `path` | `string` | The URL path that must exist in the requested URL. |
| `priority` | `string` | `High`, `Medium`, or `Low` (Chrome specific). |
| `sameSite` | `string` | `Strict`, `Lax`, or `None`. |
| `secure` | `boolean` | If `true`, the cookie is only sent over HTTPS. |

```ts
req.cookies.set("auth_token", "secret-value", {
  httpOnly: true,
  secure: true,
  sameSite: "Strict",
  maxAge: 3600, // 1 hour
  path: "/",
});
```

## Deleting cookies

To remove a cookie, call `.delete()`. Note that for a deletion to be successful, the `path` and `domain` options must match the values used when the cookie was originally set.

```ts
req.cookies.delete("session_id", { path: "/" });
```

## Manual Usage (Non-Route Handlers)

If you are using the standard `fetch` handler instead of `routes`, Bun still tracks the `CookieMap` associated with the request and applies it to the response automatically.

```ts
Bun.serve({
  fetch(req) {
    const url = new URL(req.url);
    if (url.pathname === "/set") {
      req.cookies.set("not-a-route", "true");
      return new Response("Cookie set via manual fetch handler");
    }
    return new Response("Hello");
  }
});
```

## Iterating over cookies

You can iterate over all cookies in a request using standard loop syntax:

```ts
for (const [name, value] of req.cookies) {
  console.log(`${name} is set to ${value}`);
}

// Convert all cookies to a plain object
const allCookies = Object.fromEntries(req.cookies.entries());
```
