Bun 原生支持 [Nuxt](https://nuxt.com)。使用官方 `nuxi` CLI 初始化一个 Nuxt 应用。

```sh terminal icon="terminal"
bunx nuxi init my-nuxt-app
```

---

要启动开发服务器，请从项目根目录运行 `bun --bun run dev`。这将执行 `nuxt dev` 命令（在 [package.json] 的 `"dev"` 脚本中定义）。

<Note>
  `nuxt` CLI 默认使用 Node.js；传递 `--bun` 标志会强制开发服务器使用 Bun 运行时。
</Note>

```sh terminal icon="terminal"
cd my-nuxt-app
bun --bun run dev
```

---

一旦开发服务器启动，打开 [http://localhost:3000](http://localhost:3000) 查看应用。应用将渲染 Nuxt 内置的 `NuxtWelcome` 模板组件。

要开始开发您的应用，请在 [app.vue] 中用您自己的 UI 替换 `<NuxtWelcome />`。

对于生产构建，虽然默认预设已经与 Bun 兼容，您也可以使用 [Bun 预设](https://nitro.build/deploy/runtimes/bun) 生成更优化的构建。

```ts nuxt.config.ts icon="/icons/typescript.svg"
export default defineNuxtConfig({
  nitro: {
    preset: "bun", // [!code ++]
  },
});
```

或者，您可以通过环境变量设置预设：

```sh terminal icon="terminal"
NITRO_PRESET=bun bun run build
```

<Note>
  一些包提供特定于 Bun 的导出，Nitro 在使用默认预设时无法正确打包。在这种情况下，您需要使用 Bun 预设，以便这些包在生产构建中正确工作。
</Note>

使用 bun 构建后，运行：

```sh terminal icon="terminal"
bun run ./.output/server/index.mjs
```
