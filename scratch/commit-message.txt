tls: send close_notify on end() and flush it on the duplex path (+3 tests)

Two shutdown bugs, both ending in a peer that never sees close_notify:

- Real sockets: end() did a bare TCP half-close (FIN, no close_notify) on
  the theory that BoringSSL cannot read after SSL_shutdown. It can: only
  write_shutdown is set; the read path stays open until the peer's
  close_notify arrives. Every TLS server (and client) end() looked like
  truncation ("unexpected eof") to compliant peers. end() now sends
  close_notify via ssl_handle_shutdown before the FIN, and the data path
  keeps reading with SENT_SHUTDOWN set so a TLS 1.2 write()+end() server
  still receives the peer's reply.

- Duplex path: SSLWrapper::shutdown queued close_notify into the write BIO
  but nothing pumped the memory BIO, so the alert never left the process.
  shutdown() now drains the write BIO; UpgradedDuplex drops teardown bytes
  aimed at a duplex whose write side already ended instead of surfacing a
  spurious EPIPE (TLS-inception teardown).

Fallout fixed along the way:
- A close_notify riding the same flight as the peer's Finished used to
  destroy a JS write parked on WANT_READ (written before the handshake
  finished) - retry it before the EOF close.
- Same-flight NewSessionTicket/keylog lines parked during that read are now
  delivered before the close tears the wrapper down.
- A graceful close no longer discards spilled ciphertext (deferred like the
  FAST_SHUTDOWN case, resuming with the original close code) and on_end no
  longer defeats that deferral.
- on_writable still dispatches write completions after a half-close on node
  sockets; uWS HTTP sockets keep the old suppression.
- Socket.prototype.bufferSize is undefined after close, matching node.

Adds test-tls-connect-stream-writes, test-tls-streamwrap-buffersize and
test-tls-session-timeout (all pass on node from its own tree; each fails
under a tampered assertion). Verified on the wire with openssl s_client:
close_notify present on every connection, TLS 1.2 and TLS 1.3, with
session resumption (Reused) intact.
